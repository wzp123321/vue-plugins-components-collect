# 组件规范

## 组件结构

- 组件目录在 `src/components` 下，每个组件单独一个目录
- 组件文件名为 `index.vue`，样式文件为 `index.scss` 或 `style.scss`
- 所有组件必须导入到 `src/components/index.ts` 中集中导出
- 组件必须使用 SCSS，支持全局样式和模块化样式
- 交互组件库二次封装必须使用 @tiansu/element-plus 组件，使用 te- 前缀
- 组件必须使用清晰的 TS 类型定义，Props 接口必须明确
- 使用 Vue3 Composition API 编写组件

如需查看创建组件的完整示例与拆分步骤，请使用技能文件：

- `.agents/skills/create-component/SKILL.md`

## 组件层级规划

- 单个 `.vue` 文件建议不超过 400 行（作为拆分的参考阈值）
- 遵循单一职责原则，每个组件应有明确的职责边界
- **页面级组件**：仅在当前页面使用 → 放在 `src/views/<page>/components/`
- **通用组件**：跨页面复用 → 提取到 `src/components/`

## 组件放置决策树

新增组件时，按以下流程判断放置位置：

```
组件需要在多个页面使用吗？
├─ YES → 通用组件 → 放在 `src/components/<name>/`
└─ NO → 仅在单页使用 → 放在 `src/views/<page>/components/<name>/`
```

**示例**：

- `errorBoundary`（多处使用）→ `src/components/errorBoundary/`
- `homeHeader`（仅首页使用）→ `src/views/home/components/header/`
- `craftSheetCard`（仅工艺单页面使用）→ `src/views/craftSheet/components/craftSheetCard/`

**约束**：`src/components/` 中的组件必须是真正通用的，业务耦合组件禁止放入。

## Vue3 组件规范

- 使用 `<script setup lang="ts">` 语法糖
- 组件名使用小驼峰命名法
- Props 使用 `defineProps` 定义，并指定类型
- Emits 使用 `defineEmits` 定义
- 使用 Composition API 而非 Options API
- 合理使用 `computed`、`watch`、`watchEffect`
- 组件样式使用 SCSS，支持嵌套语法

## Element Plus 组件使用规范

- 所有基础组件必须使用 @tiansu/element-plus
- 使用 te- 前缀替代 el- 前缀
- 例如：`<te-button>`、`<te-table>`、`<te-form>`
- 保持 Element Plus 的 API 一致性
- 二次封装组件时，保持 props 和 events 的一致性
